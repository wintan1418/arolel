# Security Policy

## Supported Versions

Security fixes target the `main` branch.

## Reporting a Vulnerability

Please do not open a public GitHub issue for security vulnerabilities.

Report issues by email:

`security@arolel.com`

If that address is not yet active, contact the maintainer through the GitHub repository.

Include:

- Affected feature or route
- Steps to reproduce
- Impact
- Any relevant logs or screenshots

We aim to acknowledge valid reports within 72 hours.

## Scope

In scope:

- Authentication/session issues
- Unauthorized admin access
- Server-side request or data exposure
- Stored or reflected XSS
- Secrets accidentally committed to the repository

Out of scope:

- Social engineering
- Denial-of-service testing against the public app
- Issues caused by unsupported browsers
- Vulnerabilities in third-party services outside Arolel's control
